## âš™ How It Works

### System Flow

```
      +------------------+
      |   System Start   |
      +--------+---------+
               |
               v
      +------------------+
      |  Initialize LCD  |
      |    Display:      |
      | "ENTER PASSWORD" |
      +--------+---------+
               |
               v
      +------------------+
      |  Wait for Key    |<-----------------+
      |     Press        |                  |
      +--------+---------+                  |
               |                            |
               v                            |
      +------------------+                  |
      |   Display '*'    |                  |
      |   Store digit    |                  |
      +--------+---------+                  |
               |                            |
               v                            |
         +-----------+                      |
         | 4 digits  |----No--------------->+
         | entered?  |
         +-----+-----+
               |Yes
               v
         +-----------+
         |  Compare  |
         |  Password |
         +-----+-----+
               |
       +-------+-------+
       |               |
       v               v
   +-------+       +-------+
   | Match |       |  No   |
   |       |       | Match |
   +---+---+       +---+---+
       |               |
       v               v
  +-----------+   +-----------+
  |  "ACCESS  |   |  "ACCESS  |
  |  GRANTED" |   |  DENIED"  |
  | Green ON  |   |  Red ON   |
  +-----------+   +-----------+
```

### Code Logic

1. **Initialization**: LCD and ports are initialized; LEDs are turned OFF
2. **Password Entry**: User enters 4 digits via keypad; each key displays `*`
3. **Validation**: Entered password is compared with stored password (`"1234"`)
4. **Access Granted**: If match â†’ Display "ACCESS GRANTED" â†’ Green LED ON
5. **Access Denied**: If no match â†’ Display "ACCESS DENIED" â†’ Red LED ON
6. **Reset**: After 1 second delay, system resets for next entry
